### 2. Margin Calculations - BUG FIXED

**Original Issue Found:**
The `apply_margin()` function (lines 139-193) passed spacing to `distance_transform_edt()` without reordering to match the mask array dimensions.

- **Spacing from SimpleITK**: (X, Y, Z) order
- **Mask from rt-utils**: (Y, X, Z) order (after `np.moveaxis(arr, 0, -1)` conversion)
- **distance_transform_edt requirement**: spacing must match array dimensions

**Bug:** Spacing order mismatch caused incorrect margin distances. For example, with anisotropic spacing like (0.5mm, 0.5mm, 3mm), passing X-Y-Z spacing to a Y-X-Z array would swap X and Y components.

**Fix Applied:**
Added spacing reordering before calling distance_transform_edt:
```python
# Reorder spacing to match mask array dimensions.
# Mask is (Y, X, Z), spacing from SimpleITK is (X, Y, Z).
# We need spacing in (Y, X, Z) order for distance_transform_edt.
spacing_yxz = (self.spacing[1], self.spacing[0], self.spacing[2])
```

### 3. distance_transform_edt Usage ✓ APPROVED (After Fix)

The Euclidean distance transform is correctly used:
- **Expansion**: Computes distance from background (inverted mask), includes voxels ≤ margin distance
- **Contraction**: Computes distance from foreground, keeps voxels > |margin| distance
- Uses `sampling` parameter with properly ordered spacing for anisotropic grids
- ✓ SCIENTIFICALLY CORRECT
