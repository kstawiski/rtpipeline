Knowledge Base

Research-grade documentation that keeps RTpipeline reproducible, explainable, and state-of-the-art.

Continuous Literature Review

Engineering rigor that starts with evidence

The Knowledge directory consolidates internal whitepapers on CT acquisition, DICOM-RT semantics, and modality-specific PyRadiomics settings. These documents anchor every default we ship—from voxel resampling to DVH QA thresholds—so the pipeline inherits the best practices that academic centers publish.

How the library is used

  • Translates acquisition physics research into concrete configuration templates.
  • Defines the DICOM tags RTpipeline validates before calculating DVHs or radiomics.
  • Locks PyRadiomics parameter files to IBSI-compliant settings for CT, MR, and PET.
  • Feeds automated checks surfaced in the web UI and Snakemake workflow logs.

From Knowledge to State-of-the-Art Pipeline

Insights are not static PDFs—they are encoded into validation scripts, configuration presets, and runtime safeguards.

1

Evidence Capture

Every Knowledge document cites the physics or methodology work it is based on. The repo includes the raw markdown so the rationale for each rule is always reviewable.

  • Literature findings stored in version control.
  • Change history explains why defaults evolve.
2

Encoded Defaults

The docs map directly to configuration artefacts (Snakemake params, YAML templates, Docker env vars). Updating a paper means updating code in the same pull request.

  • CT/MR/PET settings exposed as presets.
  • Metadata QA rules mirror DICOM tag guides.
3

Runtime Feedback

When RTpipeline detects a protocol drift (slice thickness, kernel, UID mismatch) it references the Knowledge topic in the log so users know the scientific reason.

  • Actionable log messages and web UI badges.
  • Guides reproducible external validation.
