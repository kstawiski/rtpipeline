Key Features

Everything you need for radiotherapy research data processing

AI-Powered Segmentation

Automatic organ segmentation using TotalSegmentator with 100+ anatomical structures. Support for custom nnU-Net models.

DVH Analysis

Comprehensive dose-volume histogram calculation with 50+ clinical metrics including Dmean, Dmax, Vx, Dx, and conformity indices.

Radiomics Extraction

IBSI-compliant feature extraction with PyRadiomics. Shape, first-order, texture, and filtered features from CT/MR images.

Batch Processing

Snakemake workflow orchestration for processing hundreds of patients. Automatic parallelization and fault tolerance.

Docker & Singularity

Pre-built containers with all dependencies. Run on local workstations, HPC clusters, or cloud platforms.

Google Colab

Run in the cloud with free GPU access. No local installation required. Perfect for testing and small datasets.

Web UI Included

The webui/ app ships with RTpipeline so researchers can manage cohorts without touching the CLI. Launch docker-compose up -d and open http://localhost:8080.

  • Drag-and-drop uploads for DICOM folders, ZIP/TAR archives, or DICOMDIR exports.
  • Automatic validation that counts patients, studies, and required RT objects before processing.
  • Collapsible configuration panes for segmentation, radiomics, robustness, CT cropping, custom nnU-Net models, and advanced flags.
  • Stage timeline + streaming logs so you can watch Organize ‚Üí Segment ‚Üí DVH ‚Üí Radiomics ‚Üí Aggregate in real time.
  • One-click ZIP download of the Data_Snakemake/<patient>/<course> tree when jobs finish.

Docker

Pre-built container with all dependencies and model weights.

# Pull the image
docker pull kstawiski/rtpipeline:latest

# Run the pipeline
docker run --gpus all \
  -v /path/to/dicom:/data/input \
  -v /path/to/output:/data/output \
  kstawiski/rtpipeline:latest \
  snakemake --cores all

Deployment Modes

Choose the workflow that matches your infrastructure

Docker Setup Wizard

Run ./setup_docker_project.sh (or the curl one-liner) to generate a project placeholder with curated docker-compose, volume layout, and helper scripts.

  • Prompts for Input/Output/Logs paths.
  • Enables CT cropping, radiomics, robustness from a guided menu.
  • Outputs project.env and run scripts you can commit.

Web UI Stack

Bring up the Flask-based UI via docker-compose up -d. GPU and CPU-only profiles are provided.

  • Mounts Input, Uploads, Output, Logs, and optional totalseg_weights.
  • NVIDIA Container Toolkit support enabled out of the box.
  • Reverse proxy friendly (nginx example in docs/nginx-rtpipeline.conf).

Snakemake CLI

For power users who prefer the terminal, clone the repo, create the Conda environments in envs/, edit config.yaml, then run Snakemake.

  • mamba env create -f envs/rtpipeline.yaml
  • snakemake --cores all --use-conda
  • rtpipeline validate checks config + env before launching.

Google Colab

Use the two official notebooks for cloud processing with free GPUs.

  • Part 1 (GPU): DICOM organise + TotalSegmentator.
  • Part 2 (CPU): DVH, radiomics, QC, aggregation.
  • Ideal for pilot studies and teaching workshops.

HPC / Singularity

Convert the Docker image with singularity pull rtpipeline.sif docker://kstawiski/rtpipeline:latest and submit Snakemake jobs to Slurm.

  • --nv flag exposes cluster GPUs.
  • Bind /data/input, /data/output, /data/logs.
  • Works with Apptainer, too.

Installation

Docker Installation

1

Prerequisites

  • Docker 20.10+
  • NVIDIA Container Toolkit (for GPU)
  • At least 32GB disk space
2

Pull the Image

docker pull kstawiski/rtpipeline:latest
3

Run the Pipeline

docker run --gpus all --rm \
  -v /path/to/dicom:/data/input:ro \
  -v /path/to/output:/data/output:rw \
  -v /path/to/logs:/data/logs:rw \
  kstawiski/rtpipeline:latest \
  snakemake --cores all --configfile /app/config.container.yaml

Conda Installation

1

Clone Repository

git clone https://github.com/kstawiski/rtpipeline.git
cd rtpipeline
2

Create Environments

mamba env create -f envs/rtpipeline.yaml
mamba env create -f envs/rtpipeline-radiomics.yaml
3

Configure and Run

cp config.yaml.example config.yaml
# Edit config.yaml with your paths

conda activate rtpipeline
snakemake --cores all --use-conda

Singularity Installation

1

Convert Docker Image

singularity pull rtpipeline.sif docker://kstawiski/rtpipeline:latest
2

Run with GPU

singularity exec --nv \
  --bind /path/to/dicom:/data/input:ro \
  --bind /path/to/output:/data/output:rw \
  rtpipeline.sif \
  snakemake --cores all --configfile /app/config.container.yaml

What You Get

Every run produces reproducible artefacts under Data_Snakemake/

Per-Course Packages

dvh_metrics.xlsx, radiomics_ct.xlsx, MR/radiomics_mr.xlsx, fractions.xlsx, QC JSON, and four RTSTRUCT variants live alongside the original DICOM.

Cohort Workbooks

_RESULTS/dvh_metrics.xlsx, radiomics_ct.xlsx, radiomics_mr.xlsx, case_metadata.xlsx, and qc_reports.xlsx consolidate everything into analysis-ready tables.

Segmentation Provenance

Manifest files track which TotalSegmentator task, nnU-Net ensemble, or manual contour contributed to each ROI so models remain auditable.

Robustness Metrics

When enabled, perturbation outputs include ICC/CoV tiers per feature plus the raw radiomics_robustness_ct.parquet for bespoke statistics.

Cropping Metadata

cropping_metadata.json captures anatomical landmarks and margins, ensuring DVH/Radiomics derived from cropped images can be justified.

Anonymisation Script

scripts/anonymize_pipeline_results.py rewrites IDs and keeps a key file, letting you share results externally without re-running the pipeline.
